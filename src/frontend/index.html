The Mathematics

Precision, formalized.

Behind the pristine UI is a battle-tested combinatorial engine. We treat operator selection as a Markov Decision Process, implementing strict Double Q-Learning to prevent overestimation bias.

\[ Y_t^{DoubleQ} \equiv R_{t+1} + \gamma Q(S_{t+1}, \mathop{\text{argmax}}_{a} Q(S_{t+1}, a; \theta_t); \theta_t^-) \]

Primary Network \(\theta_t\)

Selects the most aggressive ruin operator based on current plateau depth, time window slack, and capacity utilization. It drives the exploration phase to escape local optima.

Target Network \(\theta_t^-\)

Evaluates the actual value of the chosen action independently. This decoupling prevents the solver from getting addicted to sub-optimal shortcuts during training.
